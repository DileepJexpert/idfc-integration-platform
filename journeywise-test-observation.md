## OBS-3 — Canonical envelope on the wire: field provenance

Sample (notification `...0002`) published to `orig.sfdc.pl.v1`. Built by
`OutboundNotificationMapper` (SOAP → `SfdcInboundEvent`) then `Normalizer`
(event + routing + claim-check → `CanonicalEnvelope`).

### From the SOAP request (SFDC-supplied)
| Field | Value | Source in XML |
|---|---|---|
| `notificationId` | `04l6D00000AbCdE0002` | `<Notification><Id>` (dedup key) |
| `orgId` | `00D6D00000020HoUAI` | `<notifications><OrganizationId>` |
| `sfdcRecordId` | `a0X6D00000Rec0002` | `<sObject><sf1:Id>` |
| `type` | `Inbound_Wrapper` | `<sf1:SVCNAME__c>` (routing key) |
| `payload` | whole `createGenericAccountReq{…}` | `<sf1:Request__c>` CDATA, opaque/inline |

### Generated by the edge
| Field | Value | How |
|---|---|---|
| `transactionId` | `txn-f1b3e1d7-…` | UUID supplier in `Normalizer`, fresh per publish |
| `correlationId` | `corr-2e58…` | UUID supplier in mapper, per request, trace-only |
| `originalCorrelationId` | `corr-2e58…` | owner/first-ingest correlationId (equal here ⇒ first publish) |
| `payloadRef` | `s3://idfc-claimcheck/49497896-…` | ref from `MockS3BlobStoreAdapter.put()` (UUID at store time) |
| `occurredAt` | `2026-07-02T08:33:00.69…Z` | `clock.instant()` at ingest (NOT SFDC's body timestamps) |

### Constants / identity
| Field | Value | Source |
|---|---|---|
| `schemaVersion` | `sfdc-ingress.v1` | `Normalizer.SCHEMA_VERSION` |
| `source` | `SFDC` | `SourceSystem.SFDC` |
| `payloadContentType` | `application/json` | hardcoded in mapper |
| `applicationRef` | `null` | deliberately null — never derived from opaque body |

Subtle points:
- `occurredAt` (2026) ≠ body `msgHdr` timestamps (2023) — body is opaque.
- `correlationId == originalCorrelationId` ⇒ this is the first winning publish;
  the later Postman resend produced no new message (idempotent).
- `payload` inline **and** `payloadRef` both present = Item-11 fix (content parity).

## OBS-4 — Routing: how `type` → topic

Config-as-data in `edges/sfdc-ingress-edge/src/main/resources/application.yml`
(`idfc.edge.routing`). New business line / SVCNAME = a CONFIG ROW, not code.

| type (SVCNAME__c) | topic | downstream |
|---|---|---|
| PERSONAL_LOAN | orig.sfdc.pl.v1 | origination-journey |
| LAP | orig.sfdc.lap.v1 | origination-journey |
| BUSINESS_LOAN | orig.sfdc.bl.v1 | origination-journey |
| COMMERCIAL | orig.sfdc.commercial.v1 | origination-journey |
| **Inbound_Wrapper** | **orig.sfdc.pl.v1** | **loan-origination** (journey) |
| SENDSMS | comm.sms.send.v1 | communications (capability) |

- `Inbound_Wrapper` (CASA account creation) is mapped onto `orig.sfdc.pl.v1` as an
  end-to-end plumbing proof; the engine's `type→journey` map is empty so it resolves
  the loaded loan-origination journey. A real account-creation journey is a later,
  additive config change. (This is the known `type-to-journey` routing gap that
  workstream A must fix so account-creation stops running the origination DAG.)
- `known-orgs` allow-list includes the golden fixture org `00D6D00000020HoUAI`;
  unknown org → refresh + recheck → DLQ (C2).

## OBS-5 — Claim-check body ("open the payloadRef")

- `payloadRef = s3://idfc-claimcheck/<uuid>` is **in-memory only**
  (`MockS3BlobStoreAdapter`, `ConcurrentHashMap`) — not a file/URL.
- The stored bytes are byte-identical to the inline `payload`. Parity oracle (§F)
  resolves the ref and compares bytes; that equality is the Item-11 content guarantee.
- A real S3 adapter swaps in behind `BlobStorePort` with no domain change.
