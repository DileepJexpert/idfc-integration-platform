Gate before building — confirm with IDFC (flagged, not bucketed)

◆ Pivotal — LMS: facade or per-endpoint?
The catalog shows LMS as per-endpoint (distinct paths), but lms-utilities was built as a single facade (one /callLmsUtilities, requestCode in the body). They contradict. The answer sets the capability shape for ~19 svcNames — settle it first.
⚠ Foreclosure — read or execute?
Leaned read (existing lending-servicing models FinnOne foreclosure as a quote/amount read — "servicing never books"), but the verb could be execute. If it's an action mislabeled retry-safe, a retried 5xx could double-execute a foreclosure.
⚠ IMPS_TO_VALIDATE_CUSTOMER — penny-drop?
A "validate" is usually a read, but /P2A may move ₹1 to validate an account → then it's an action needing idempotency, not a read. Same double-execution risk if misclassified.
⚠ M (mappers) — needs payloads
Each read is T1 only if its response is the shared house envelope; otherwise T2. M is the difference between a cheap and a moderate migration and can't be closed without sample responses. Don't promise the zero-code number yet.
